Dưới đây là **checklist rà soát hình thức trình bày và logic bài báo AI/NLP**. Khung này có thể dùng như **pre-submission checklist** trước khi gửi hội nghị hoặc journal. Các tiêu chí tập trung vào:

1. **format và layout**
2. **logic lập luận**
3. **tính nhất quán nội dung**
4. **chất lượng bảng hình và số liệu**
5. **tính chuẩn hóa học thuật**

Checklist này độc lập với nội dung kỹ thuật, dùng để phát hiện **lỗi trình bày mà reviewer thường bắt ngay khi đọc bài**.

---

# I. Format tổng thể bài báo

| Rule | Kiểm tra                                                                                   |
| ---- | ------------------------------------------------------------------------------------------ |
| F1   | Title rõ ràng, phản ánh đúng phương pháp và task                                           |
| F2   | Abstract chứa đủ: problem – method – dataset – result                                      |
| F3   | Keywords phù hợp với lĩnh vực NLP/AI                                                       |
| F4   | Section numbering nhất quán                                                                |
| F5   | Tên section ngắn gọn và đúng chuẩn (Introduction, Related Work, Methodology, Experiments…) |
| F6   | Không có section thừa hoặc thiếu                                                           |
| F7   | Section order hợp lý                                                                       |
| F8   | Font và spacing thống nhất                                                                 |
| F9   | Không có paragraph quá dài                                                                 |
| F10  | Không có câu quá dài hoặc nhiều mệnh đề                                                    |

---

# II. Logic của bài báo

| Rule | Kiểm tra                                        |
| ---- | ----------------------------------------------- |
| L1   | Introduction dẫn từ bối cảnh → bài toán → gap   |
| L2   | Research gap được nêu rõ                        |
| L3   | Motivation của phương pháp được giải thích      |
| L4   | Contribution được liệt kê rõ                    |
| L5   | Methodology trả lời trực tiếp research question |
| L6   | Experiments kiểm chứng contribution             |
| L7   | Discussion giải thích kết quả                   |
| L8   | Conclusion tóm tắt đúng contribution            |

---

# III. Logic giữa các section

| ---- | ----------------------------------------------- |

# XVII. Checklist reviewer đọc nhanh (5 phút)

Một reviewer thường kiểm tra nhanh:

1

Abstract có số liệu kết quả không

2

Contribution có rõ không

3

Experiments có baseline mạnh không

4

Result improvement có đáng kể không

5

Discussion có giải thích kết quả không

---

# XVIII. Lỗi hình thức reviewer thường bắt ngay

Các lỗi phổ biến:

1

Citation sai format

2

Table không được nhắc trong text

3

Figure caption quá ngắn

4

Abstract không có số liệu

5

Method thiếu motivation

---

# XIX. Checklist tự rà trước khi nộp

Trước khi submit, tác giả nên kiểm tra:

* logic bài
* consistency số liệu
* format citation
* reproducibility
* clarity.
